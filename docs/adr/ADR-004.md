# ADR-004 — Workcell-Grenze: Materials, Artifacts & Secret-Redaction

**Status:** akzeptiert · **Phase:** 1 (+ Ausblick Phase 3) · **baut auf** ADR-001, ADR-002, ADR-003

## Kontext
Die Workcell-Grenze ist der Punkt, an dem Materials rein-, Artifacts raus- und Secrets durchqueren. Der Cell-Shim (ADR-001) sitzt genau auf dieser Grenze und ist damit der einzige richtige Ort für Materialisierung, Artefakt-Externalisierung und Redaction — bevor irgendetwas append-only wird (ADR-002).

## Entscheidung 1 — Materials rein: **snapshot-first, mount-optional (Policy)**
Keine starre Methode, sondern eine `materialization_mode`-Policy:
- **canonical / reproduzierbar:** Snapshot in die Workcell kopieren.
- **fast / iterativ:** read-only Bind-Mount / Live-Mount.
- **explizite Dev-Modi:** echte Live-Bind-Semantik.

Phase 1: produktionsnahe Runs → **Snapshot**; lokale/vertrauenswürdige Runs → optional Mount; später Hybrid-Policy. Snapshot ist die reproduzierbare Baseline; Mount erzeugt sonst Isolations-/Drift-Probleme.

**Seed (Amendment zu ADR-001):** jede Session hält `materialization_mode` + `workbench_snapshot_ref` **oder** `workbench_mount_ref` fest.

## Entscheidung 2 — Artifacts raus: **content-addressed Store, Events nur Pointer**
Sehr entschieden: **Events enthalten nie die eigentlichen Inhalte.** Der Event-Stream bleibt das chronologische Wahrheitssignal, nicht der Payload-Speicher.

`worker.artifact_written` trägt nur Metadaten + Pointer: `path · content_hash · size · mime_type · artifact_ref`. Blobs/Objekte liegen im separaten **Artifact Store** (content-addressed).

**Warum:** Stream bleibt schlank, ewig speicherbar, replayfähig, auditierbar.

## Entscheidung 3 — Secrets über die Grenze: **Redaction im Cell-Shim, nie im dauerhaften Stream**
Harte Regel: Secrets landen **nie** in `worker.raw`, nicht im Controller, nicht im persistenten Event-Log, nicht in Artefakten, nicht im UI.

**Sicherheitsmodell (Phase-1-Minimalpfad):**
- Secret als Capability / ephemeres Handle in die Workcell → Worker erhält es als Env oder Temp-File.
- Shim hält eine **Redaction-Liste**; alles (stdout/stderr/File-Events) wird **vor Persistenz** gescannt.
- Treffer → ersetzt durch `[REDACTED]`.
- Zusätzlich Sicherheits-Events: `security.secret_leak_detected`, `security.redaction_applied`.

**Kernpräzisierung:** `worker.raw` ist eigentlich **durable sanitized raw**. transient raw (flüchtig im Shim) vs. persisted raw (nur nach Redaction). „Fidelity geht nie verloren" (ADR-002) gilt **mit genau einer bewussten Ausnahme: Secrets**.

## Nachträgliche Amendments (via ADR-006/007)
- **E3 abgelöst durch Secret-Broker (ADR-006 E5):** Die statische Env-Injection dieses ADR ist der Phase-1-Minimalpfad. Ab Phase 3 mintet ein **Secret-Broker** (control-plane, getrennt vom Controller) per-Run kurzlebige, gescopte Capability-Handles/dynamische Creds (Injection env|tmpfs, Rotation mid-run, Leases überleben Suspension, Revocation bei reap). **Die Redaction dieses ADR bleibt unverändert die Persistenz-Wache** — der Broker registriert injiziertes Material automatisch in die Redaction-Liste. Neue Secret-Events: `security.secret_leased/rotated/revoked` (nur Metadaten).
- **E1 präzisiert (ADR-006 E4):** `mount` (writable Live-Bind) nur unter `isolation_profile = standard` + trusted; unter `hardened`/`strict` fail-closed auf Snapshot gezwungen (read-only nur via virtio-fs).
- **E2/CAS (ADR-007):** Artifact Store teilt das content-addressed CAS-Substrat mit dem Warehouse (getrennter Katalog). `materialization_mode == mount`-Runs sind explizit **nicht** bit-reproduzierbar. GC zählt gegen die Vereinigung aus Run-Seeds, Channel-Pointern, Event-Stream-Pointern (`artifact_ref`/`worker.raw`) und Artifact-Manifest.
