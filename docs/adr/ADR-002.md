# ADR-002 — Event-Sourcing als Fundament & kanonisches Event-Schema

**Status:** akzeptiert · **Phase:** 1 · **baut auf** ADR-001

## Kontext
Der Cell-Shim (ADR-001) befüllt einen Event-Stream; der Controller sieht diesen Stream als einzige Wahrheit. Es braucht eine gemeinsame, stabile Sprache dafür, an der Lifecycle-State-Machine (Phase 2), Gate-Mechanik und Artifact-Manifest hängen.

## Entscheidung — Event-Sourcing als Grundprinzip
- **Der Event-Stream *ist* der Run**, kein Nebenprodukt. Terminal-State, Artifact-Manifest und Reproduzierbarkeits-Seeds (ADR-001) sind vollständig aus dem Stream rekonstruierbar.
- **Controller = Projektion** über dem Log, kein zweiter mutable State-Speicher. Vorteil: sauberes Debugging, Replay, Audit.
- **append-only**, `run_id` + **monotone `seq`** pro Run.
- **`worker.raw` wird nie verworfen.** Alles, was ein (dünner) Adapter nicht mappen kann, bleibt als Raw-Event erhalten → Fidelity geht nie verloren. *(Präzisierung via ADR-004: gilt mit genau einer bewussten Ausnahme — redigierte Secrets.)*

## Event-Taxonomie (vier Klassen + Fallback)
```
lifecycle:  run.created · run.provisioning · run.started · run.finished
semantic:   worker.message · worker.tool_call · worker.tool_result · worker.artifact_written
control:    gate.requested · gate.resolved · run.paused · run.resumed · run.cancel_requested
health:     heartbeat · resource.sample · error
security:   security.secret_leak_detected · security.redaction_applied   (via ADR-004)
fallback:   worker.raw   (nie verwerfen — durable sanitized raw)
```

## Event-Envelope
**Pflicht ab Tag 1 (jedes Event):** `run_id` · `seq` (monoton) · `schema_version` (pro Event, nicht nur global) · `timestamp` · `type` · `payload`.
**Optional:** `source`/`emitter` · `correlation_id` · `causation_id`.

## Payload-Regel (Amendment via ADR-004)
Events tragen **nie große Inhalte**, nur Metadaten + Pointer. Datei-/Artefakt-Inhalte liegen im content-addressed Artifact Store; das Event referenziert per `content_hash` / `artifact_ref`.

## `worker.raw` = durable sanitized raw (Amendment via ADR-004)
- **transient raw** — ungefiltert, existiert höchstens flüchtig im Shim-Speicher.
- **persisted raw** — nur **nach** Secret-Redaction append-only.

Fidelity geht nie verloren — außer für Secrets, deren Weglassen Absicht ist.

## Konsequenzen
- Neue Event-Typen sind additiv; alte Runs bleiben lesbar (dank `schema_version` je Event + `worker.raw`).
- Jede spätere Schicht (Gates, Warehouse, Outer-Loop-Consumer) konsumiert denselben Stream statt eigener Zustände.
- *(Amendment via ADR-008):* Der append-only Event-Stream ist zugleich eine erstklassige **externe Contract-Fläche** mit definierter Delivery-Semantik (at-least-once, seq-idempotent, Replay ab beliebigem `from_seq`, keine Consumer-Backpressure auf die Execution); `schema_version` pro Event ist die Forward-Compat-Achse für externe Consumer.
