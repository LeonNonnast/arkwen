# ADR-003 — Run Lifecycle & Artifact Projection

**Status:** akzeptiert · **Phase:** 1→2 · **baut auf** ADR-001, ADR-002

## Kontext
Der Event-Stream ist die Wahrheit (ADR-002). Lifecycle-State und Artefakte sind daher keine eigenen Wahrheitsspeicher, sondern **Projektionen** (Folds) über dem Stream. Dieser ADR fixiert die Zeit-Schicht (Zustände + Unterbrechungen) und die Raum-Schicht (Artefakte, Diffs, Checksummen).

## Teil A — Run Lifecycle (Zeit-Schicht)
Zwei orthogonale Ebenen statt eines flachen State-Sets:

**1. Kanonischer Top-Level-State**
`queued · provisioning · running · finalizing · completed · failed · canceled`
- Terminal (kein Overlay mehr): `completed`, `failed`, `canceled`.

**2. Orthogonales Suspension-Overlay** (`suspension_reason`)
`none · paused_by_gate · paused_by_user · paused_by_policy · paused_by_resource_limit`

**Prinzip:** `paused` ist **kein** eigener harter Zustand, sondern ein Overlay über `running`. Ein Run ist fachlich nicht „weg", wenn ein Gate greift — er ist suspendiert und wartet auf eine Entscheidung.
- `running + paused_by_gate` = aktiv, aber wegen Gate suspendiert.
- `running + none` = normal aktiv.
- `finalizing + none` = Ergebnis wird materialisiert.

Vorteil ggü. eigenem `paused`-Top-State: das *Warum* der Pause muss nie geraten werden.

**Gate-Verhalten (Event-Fluss):**
```
gate.requested  →  [run.paused]  →  gate.resolved  →  [run.resumed]
```
`run.paused` / `run.resumed` bleiben Control-Events im Stream; der tatsächliche Zustand entsteht aus der Projektion.

## Teil B — Artifact Projection (Raum-Schicht)
Der Controller foldet den Stream zu drei Projektionen (eine vierte kommt via ADR-008 hinzu):

**1. Artifact Manifest** — aus `worker.artifact_written`, `worker.tool_result`, dateibezogenen `worker.raw`. Felder je Artefakt:
`artifact_id · path · content_hash · mime_type · size · created_by · created_at_seq · source_event_seq`

**2. Workbench Diff** — Delta zwischen Baseline (`workbench_base_commit` + `workbench_snapshot_ref`) und rekonstruiertem Stand.

**3. Renderbare Run-Summary** — was das UI zeigt: Modified files · Tests · Summary · Gate-Status · final artifact set.

**4. Run-Metrics-Projektion** *(Amendment via ADR-008)* — cost · duration (inkl. `suspended_ms_by_reason`) · gate_outcomes · terminal · artifact_signals. Reiner Fold, best-effort, tier-degradierend.

## Prinzipien
- Alles **content-addressed**, alles **aus Events projizierbar**.
- Baseline **immer explizit**; `worker.raw` darf auch hier nie verloren gehen.
- Terminal-Zustände sind nur `completed · failed · canceled`.

## Amendment zu ADR-001
Seed-Set wird erweitert um **`workbench_snapshot_ref`** (Baseline-Hash / echter Workspace-Snapshot), zusätzlich zu `workbench_base_commit` — damit Reproduzierbarkeit auf tatsächlicher Workspace-Identität fußt, nicht nur auf Git-Commit-Ebene.
